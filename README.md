## Описание реализации

Это моя реализация тестового задания.

Когда пользователь заходит в приложение, данные по нашему API загружаются только **один раз**.  
Это сделано потому, что дальше данные для **избранного** должны браться из **базы данных**.  
Если бы запрос выполнялся каждый раз при открытии экрана курсов, это по логике обновляло бы базу данных,  
а также мешало бы корректному отображению изменений в избранном.  
Поэтому данные из API вызываются только один раз.

---

### Детали реализации

- Добавил **красный значок соединения**, чтобы было видно, что данные берутся из базы данных.  
- Реализовал **прогресс-бар**, **обработку ошибок** и **экран деталей** — это, скорее, для себя.  
- Картинку использовал **базовую везде**, так как в API ничего такого нет.  
- Данные из **сети**, **доменного слоя** и **базы данных** мапятся по структуре «звезды».
